# Product Overview

## 製品概要

Kiro形式のスペック駆動開発フレームワーク - Claude Codeのスラッシュコマンド、フック、エージェントを活用した構造化された開発ワークフローを提供します。

## コア機能

### スペック駆動開発ワークフロー
- **フェーズ0: ステアリング（オプション）**: プロジェクト全体のルールとコンテキストを定義
  - `/kiro:steering` - ステアリングドキュメントの作成・更新
  - `/kiro:steering-custom` - 特化したコンテキスト向けのカスタムステアリング

- **フェーズ1: 仕様作成**: 3段階の承認ワークフロー
  - `/kiro:spec-init` - 詳細なプロジェクト説明から仕様を初期化
  - `/kiro:spec-requirements` - 要件ドキュメントの生成
  - `/kiro:spec-design` - 技術設計の作成（要件レビュー必須）
  - `/kiro:spec-tasks` - 実装タスクの生成（設計レビュー必須）

- **フェーズ2: 実装とトラッキング**:
  - `/kiro:spec-impl` - TDD手法による実装
  - `/kiro:spec-status` - 進捗状況の確認
  - `/kiro:validate-design` - 設計品質レビュー
  - `/kiro:validate-gap` - 要件と実装のギャップ分析

### GitHub自動化機能
- **Issue重複検出**: `/dedupe` コマンドによる重複Issueの自動検出
- **自動クローズスクリプト**: 3日間未反応の重複Issueを自動クローズ
- **PR作成**: `/commit-push-pr` によるコミット・プッシュ・PR作成の自動化

### 拡張性とカスタマイズ
- **Hooksシステム**: Pre/PostToolUseフックによるコマンド実行の制御
- **カスタムスラッシュコマンド**: マークダウンベースのコマンド定義
- **エージェント活用**: 複雑なタスクを自律的に実行

## ターゲットユースケース

### 主要ユースケース
1. **構造化された機能開発**: 要件→設計→タスク→実装の段階的ワークフロー
2. **品質保証**: TDD手法による実装と設計検証
3. **GitHub Issue管理**: 重複Issueの検出と自動クローズ
4. **開発自動化**: コミット・PR作成の効率化

### 適用シナリオ
- 新機能の追加時に仕様を段階的に作成
- 既存コードベースへの小規模な追加（spec-init直接実行可）
- GitHubリポジトリのIssue管理自動化
- チーム開発での一貫性のある開発プロセス

## キーバリュープロポジション

### 独自の利点
1. **段階的な承認プロセス**: 各フェーズで人間によるレビューを必須化
2. **スキップ不可のワークフロー**: 設計は承認済み要件を、タスクは承認済み設計を必須化
3. **TDD統合**: Kent BeckのTDD手法を標準実装プロセスに組み込み
4. **Claude Codeネイティブ**: スラッシュコマンドとフックによるシームレスな統合
5. **柔軟なステアリング**: プロジェクト全体またはファイルパターン別のコンテキスト管理

### 差別化要素
- マークダウンベースの宣言的なコマンド定義
- ステアリングドキュメントによるAIへのプロジェクト知識の注入
- GitHub APIとの統合による自動化スクリプト
- 日本語での思考・応答生成対応

## 開発ルール
- 英語で思考し、日本語で応答を生成
- 各フェーズで人間の承認を必要とする
- フェーズのスキップは不可
- タスク完了時のステータス更新を必須化
- 重大な変更後はステアリングを更新
