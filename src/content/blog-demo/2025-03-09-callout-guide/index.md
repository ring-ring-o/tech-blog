# Callout - 補足情報ブロックの使い方

この記事では、ブログ記事内で補足情報を強調表示するための**Callout**記法について解説します。

## Calloutとは

Calloutは、記事内で重要な情報、警告、ヒントなどを目立たせるためのマークダウン拡張記法です。5種類のタイプがあり、それぞれ異なる色とアイコンで表示されます。

## 基本的な書き方

開始タグ（`:::type`）、内容、終了タグ（`:::`）をそれぞれ空行で区切って記述します。

## タイプ別の使用例

### note - 補足情報

一般的な補足情報や追加説明に使用します。

:::note

これは補足情報です。読者に追加のコンテキストを提供したいときに使用します。

:::

タイトルを付けることもできます。

:::note 環境について

この記事のコード例はNode.js v20で動作確認しています。

:::

### tip - ヒント・コツ

便利なヒントやベストプラクティスを紹介するときに使用します。

:::tip

`console.log`の代わりに`console.table`を使うと、オブジェクトやオブジェクト配列が表形式で見やすく表示されます。

:::

:::tip パフォーマンスのコツ

大量のDOM操作を行う場合は、`DocumentFragment`を使用してバッチ処理することで、再描画の回数を減らせます。

:::

### warning - 警告

注意が必要な情報や、間違いやすいポイントを警告するときに使用します。

:::warning

この機能は実験的なものです。本番環境での使用は推奨されません。

:::

:::warning 非推奨の機能

`componentWillMount`は非推奨です。代わりに`useEffect`または`componentDidMount`を使用してください。

:::

### caution - 注意事項（危険）

危険な操作や、データ損失の可能性がある場合に使用します。

:::caution

この操作は元に戻せません。実行前に必ずバックアップを取ってください。

:::

:::caution セキュリティ上の注意

ユーザー入力を直接SQLクエリに埋め込むと、SQLインジェクション攻撃のリスクがあります。必ずプリペアドステートメントを使用してください。

:::

### important - 重要な情報

特に重要な情報や、必ず読んでほしい内容に使用します。

:::important

このライブラリを使用するにはAPIキーが必要です。APIキーは環境変数`API_KEY`に設定してください。

:::

:::important 破壊的変更

v3.0ではAPIが大幅に変更されました。v2からの移行には移行ガイドを参照してください。

:::

## 複数段落のコンテンツ

Callout内に複数の段落を含めることもできます。

:::note 複数段落の例

2つ目の段落です。空行で区切ることで、別の段落として認識されます。

3つ目の段落です。必要に応じていくつでも段落を追加できます。

:::

## リストを含むコンテンツ

箇条書きや番号付きリストも使用できます。

:::important 必要な準備

開発を始める前に、以下のツールをインストールしてください:

- Node.js v20以上
- pnpm（パッケージマネージャー）
- Git
- Docker（オプション）

:::

1. まずJavaScriptの基礎を固める

## 使用上の注意

:::warning 記法のルール

Calloutを正しく表示するには、以下のルールを守ってください:

- 開始タグ（`:::type`）の後に空行を入れる
- 終了タグ（`:::`）の前に空行を入れる
- 開始タグと終了タグは単独の行に記述する

:::

## まとめ

Calloutを活用することで、読者にとって重要な情報を効果的に伝えることができます。

:::tip

適切なタイプを選んで使用することで、記事の可読性が向上します。

:::
